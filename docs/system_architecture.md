# 🏗️ معماری کلان سیستم (System Architecture) - Fistap

## ۱. مدل ارتباطی
سیستم از مدل **Hybrid Communication** استفاده می‌کند:
- **HTTP REST API:** برای عملیات‌های سنگین و تک‌باره (ثبت‌نام، آپلود فایل، تغییر پروفایل).
- **WebSockets (Socket.io):** برای تمام ارتباطات لحظه‌ای (ارسال پیام، اعلان‌ها، وضعیت تایپ).

## ۲. جریان داده (Data Flow)
1. **اتصال:** کاربر از طریق PWA متصل شده و با JWT احراز هویت می‌شود.
2. **ارسال پیام:** پیام از طریق Socket.io به سرور ارسال $\rightarrow$ ذخیره در MongoDB $\rightarrow$ ارسال سریع به گیرنده (اگر آنلاین است) $\rightarrow$ در غیر این صورت ارسال Push Notification.
3. **مدیریت وضعیت:** وضعیت آنلاین بودن کاربر در Redis ذخیره می‌شود تا فشار روی MongoDB کاهش/ شود.

## ۳. استراتژی دسترسی بدون VPN
- استفاده از سرورهای لبه (Edge Servers) در نقاط استراتژیک.
- بهینه‌سازی MTU و TCP برای کاهش احتمال قطعی در شبکه‌های داخلی.
